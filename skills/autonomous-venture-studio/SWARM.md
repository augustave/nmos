# Venture Studio Swarm

This swarm defines a reciprocal-feedback operating model rather than a linear
handoff chain. Each node owns a different truth function.

## Node 1: Product Foundry Architect

Responsibilities:

- convert founder ambiguity into job-to-be-done clarity
- define domain model, entities, workflows, and permissions
- prototype the core utility with enterprise-grade complexity support

Outputs:

- `validated_market_requirement_v1`
- `domain_model_v1`
- `workflow_state_spec_v1`
- `interactive_prototype_v1`

## Node 2: Brand Experience Sovereign

Responsibilities:

- define the aesthetic and verbal system
- ensure product and GTM share one design DNA
- create motion, campaign, and brand rules that scale

Outputs:

- `visual_identity_tokens_v1`
- `motion_language_v1`
- `brand_operating_system_v1`
- `launch_campaign_architecture_v1`

## Node 3: Strategic Narrative Operator

Responsibilities:

- translate product and brand truth into executive clarity
- resolve cross-node conflicts
- govern autonomy, evidence, and launch readiness

Outputs:

- `executive_narrative_v1`
- `design_ops_playbook_v1`
- `governance_evidence_vault_v1`

## Review Gates

### Market Truth Gate

Pass only if:

- target user is named
- pain is documented
- the core wedge is specific

### Systems Truth Gate

Pass only if:

- domain model exists
- workflow states are explicit
- prototype reflects real product logic

### Aesthetic Coherence Gate

Pass only if:

- product and campaign feel like the same company
- visual system is distinctive and reusable

### Narrative Truth Gate

Pass only if:

- executive story maps to actual or disclosed capability
- product and brand claims do not outrun reality

### Governance Gate

Pass only if:

- assumptions are recorded
- risk tiering is explicit
- overrides and disclosures are documented

## Venture Loop

1. Narrative operator defines or sharpens the market requirement.
2. Product foundry converts it into a domain model and prototype.
3. Brand sovereign turns the product system into a differentiated market object.
4. Narrative operator reconciles promise, capability, and launch readiness.
5. The swarm either greenlights the next phase or routes back to the blocking node.
